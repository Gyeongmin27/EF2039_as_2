# 제품 요구사항 문서 (PRD)
## 동체시력 테스트 웹 게임

---

## 1. 제품 개요

### 1.1 제품 정의
동체시력 테스트 웹 게임은 사용자의 동체시력(움직이는 물체를 인식하고 추적하는 능력)을 테스트하고 향상시키기 위한 교육용 웹 애플리케이션입니다. 화면의 여러 위치에서 동시에 떨어지는 한글 글자를 인식하고 기억하여 단어를 완성하는 게임 형식으로 제공됩니다.

### 1.2 타겟 오디언스
- **주요 타겟**: 학생, 일반 성인 (연령 10세 이상)
- **사용 목적**: 
  - 동체시력 향상 훈련
  - 인지 능력 테스트
  - 재미있는 두뇌 게임 경험
- **기술 수준**: 기본적인 웹 브라우저 사용 능력

### 1.3 핵심 목적
사용자가 빠르게 움직이는 여러 글자를 동시에 인식하고 기억하여 단어를 완성하는 능력을 테스트하고 향상시키는 것입니다. 이를 통해 집중력, 기억력, 동체시력을 종합적으로 향상시킬 수 있습니다.

---

## 2. 프로젝트 목표

### 2.1 사용자 목표
- 동체시력을 재미있게 테스트하고 향상시킬 수 있다
- 난이도를 조절하여 자신의 수준에 맞게 게임을 즐길 수 있다
- 점수를 통해 자신의 실력을 확인하고 개선할 수 있다

### 2.2 비즈니스 목표
- 교육용 웹 애플리케이션의 기본 구조와 사용자 경험을 제공한다
- HTML, CSS, JavaScript를 활용한 프론트엔드 개발 실습을 제공한다
- 제품 기획부터 구현까지의 전체 프로세스를 경험한다

### 2.3 기술 목표
- 순수 웹 기술(HTML, CSS, JavaScript)만으로 완전한 게임을 구현한다
- 반응형 디자인을 적용하여 다양한 화면 크기에서 작동한다
- 깔끔하고 직관적인 사용자 인터페이스를 제공한다

## 3. 기능 목록

### 3.1 Must-have 기능 (필수)

#### 3.1.1 게임 시작 및 단어 생성
- **우선순위**: Must-have
- **설명**: 랜덤으로 2~3글자의 한글 단어를 선택하여 게임에 사용
- **세부 요구사항**:
  - 한글 단어 데이터베이스 구축 (2글자, 3글자 단어 각각 최소 20개 이상)
  - 랜덤 선택 알고리즘 구현
  - 선택된 단어의 각 글자를 개별적으로 처리

#### 3.1.2 난이도 설정 시스템
- **우선순위**: Must-have
- **설명**: 글자 획수에 따라 난이도를 상, 중, 하로 구분
- **세부 요구사항**:
  - 난이도 상: 복잡한 글자(획수 10 이상)
  - 난이도 중: 중간 복잡도 글자(획수 5-9)
  - 난이도 하: 간단한 글자(획수 4 이하)
  - 사용자가 난이도를 선택할 수 있는 UI 제공

#### 3.1.3 글자 떨어뜨리기
- **우선순위**: Must-have
- **설명**: 화면 위쪽에서 글자가 떨어지는 애니메이션
- **세부 요구사항**:
  - 각 글자는 화면 위쪽의 중앙 주변 랜덤 위치에서 시작 (가장자리 제외: 화면 너비의 20%~80% 범위)
  - 중력 효과를 적용한 자연스러운 낙하 애니메이션
  - 글자들이 랜덤 순서로 떨어짐
  - 글자가 화면 하단에 도달하면 사라짐

#### 3.1.4 떨어지는 속도 조절
- **우선순위**: Must-have
- **설명**: 사용자가 글자가 떨어지는 속도를 조절할 수 있는 기능
- **세부 요구사항**:
  - 속도 조절 슬라이더 또는 버튼 제공
  - 최소 속도(느림)부터 최대 속도(빠름)까지 3-5단계 제공
  - 실시간으로 속도 변경 가능

#### 3.1.5 입력 시스템
- **우선순위**: Must-have
- **설명**: 모든 글자가 떨어진 후 사용자가 단어를 입력하는 기능
- **세부 요구사항**:
  - 입력창 표시
  - 1.5초 시간 제한 타이머 표시 (0.1초 단위로 업데이트)
  - 입력 완료 버튼 또는 엔터 키로 제출
  - 시간 초과 시 자동 제출

#### 3.1.6 점수 시스템
- **우선순위**: Must-have
- **설명**: 정답 입력 시 점수 증가 및 자동 다음 라운드 진행, 오답 시 게임 종료
- **세부 요구사항**:
  - 정답 입력 시 점수 증가 (난이도에 따라 다른 점수 부여)
  - 정답 시 결과 화면 없이 자동으로 다음 라운드 진행
  - 오답 입력 시 획득한 점수를 표시한 후 처음 화면으로 이동
  - 현재 점수 실시간 표시

#### 3.1.7 게임 재시작 기능
- **우선순위**: Must-have
- **설명**: 게임 종료 후 다시 시작할 수 있는 기능
- **세부 요구사항**:
  - "다시 시작" 버튼 제공
  - 오답 시 자동으로 처음 화면으로 복귀

### 3.2 Should-have 기능 (권장)

#### 3.2.1 게임 화면 전환
- **우선순위**: Should-have
- **설명**: 메인 화면, 게임 화면, 설정 화면 간의 부드러운 전환
- **세부 요구사항**:
  - 페이지 간 네비게이션 메뉴
  - 전환 애니메이션 효과

#### 3.2.2 시각적 피드백
- **우선순위**: Should-have
- **설명**: 사용자 액션에 대한 시각적 피드백 제공
- **세부 요구사항**:
  - 정답 시 성공 메시지 및 효과
  - 오답 시 실패 메시지 및 효과
  - 타이머 시각적 표시 (진행 바 또는 숫자)

#### 3.2.3 반응형 디자인
- **우선순위**: Should-have
- **설명**: 다양한 화면 크기에 대응하는 레이아웃
- **세부 요구사항**:
  - 데스크톱, 태블릿, 모바일 화면 지원
  - 미디어 쿼리를 활용한 반응형 CSS

### 3.3 Nice-to-have 기능 (선택)

#### 3.3.1 사운드 효과
- **우선순위**: Nice-to-have
- **설명**: 게임 액션에 대한 사운드 효과
- **세부 요구사항**:
  - 정답/오답 사운드
  - 배경 음악 (선택 가능)

#### 3.3.2 통계 기능
- **우선순위**: Nice-to-have
- **설명**: 게임 플레이 통계 기록
- **세부 요구사항**:
  - 최고 점수 기록
  - 평균 정답률
  - 플레이 횟수

## 4. 기술 및 디자인 요구사항

### 4.1 기술 스택

#### 4.1.1 프론트엔드
- **HTML5**: 웹 페이지 구조 및 시맨틱 마크업
- **CSS3**: 스타일링, 애니메이션, 반응형 디자인
- **JavaScript (ES6+)**: 게임 로직, 애니메이션 제어, 사용자 인터랙션

#### 4.1.2 추가 라이브러리/프레임워크
- **선택사항**: 순수 JavaScript로 구현하되, 필요시 다음을 고려
  - CSS 프레임워크: 부트스트랩 또는 Tailwind CSS (선택)
  - 애니메이션: CSS 애니메이션 또는 Web Animations API
  - 폰트: Google Fonts (한글 폰트)

#### 4.1.3 개발 도구
- **버전 관리**: Git, GitHub
- **코드 에디터**: Visual Studio Code 또는 유사한 IDE
- **브라우저 호환성**: Chrome, Firefox, Safari, Edge (최신 버전)

### 4.2 UI/UX 가이드라인

#### 4.2.1 디자인 원칙
- **단순성**: 직관적이고 이해하기 쉬운 인터페이스
- **명확성**: 모든 버튼과 기능의 목적이 명확함
- **일관성**: 전체 페이지에 걸쳐 일관된 디자인 언어 사용

#### 4.2.2 색상 팔레트
- **주요 색상**: 
  - 배경: 밝은 회색 또는 흰색 (#F5F5F5 또는 #FFFFFF)
  - 주요 액션: 파란색 또는 초록색 (#4A90E2 또는 #52C41A)
  - 경고/오류: 빨간색 (#FF4D4F)
  - 텍스트: 진한 회색 또는 검은색 (#333333 또는 #000000)
- **대비**: WCAG 2.1 AA 기준 준수 (최소 4.5:1)

#### 4.2.3 타이포그래피
- **한글 폰트**: Noto Sans KR 또는 나눔고딕
- **영문 폰트**: 시스템 폰트 스택 사용
- **폰트 크기**: 
  - 제목: 24px 이상
  - 본문: 16px
  - 작은 텍스트: 14px

#### 4.2.4 레이아웃
- **페이지 구조**: 
  - 헤더: 게임 제목 및 네비게이션
  - 메인 콘텐츠 영역: 게임 화면
  - 푸터: 간단한 정보
- **그리드 시스템**: Flexbox 또는 CSS Grid 활용

#### 4.2.5 인터랙션 디자인
- **버튼**: 
  - 최소 크기: 44x44px (터치 친화적)
  - 호버 효과: 색상 변화 또는 그림자
  - 클릭 피드백: 시각적 변화
- **입력 필드**: 
  - 명확한 테두리
  - 포커스 상태 표시
  - 플레이스홀더 텍스트

### 4.3 성능 요구사항
- **로딩 시간**: 초기 페이지 로드 2초 이내
- **애니메이션**: 60fps 유지 (부드러운 애니메이션)
- **반응성**: 사용자 액션에 대한 응답 시간 100ms 이내

### 4.4 접근성 요구사항
- **키보드 네비게이션**: 모든 기능을 키보드로 접근 가능
- **스크린 리더**: 시맨틱 HTML 태그 사용
- **색상 의존성**: 색상만으로 정보를 전달하지 않음

## 6. 제약사항 및 가정

### 6.1 제약사항
- **브라우저**: 최신 브라우저만 지원 (IE 미지원)
- **인터넷 연결**: 오프라인에서도 작동해야 함 (CDN 사용 최소화)

- 사용자는 한글을 읽고 입력할 수 있음
- 사용자는 기본적인 웹 브라우저 사용법을 알고 있음
- 화면 크기는 최소 320px 너비 이상
